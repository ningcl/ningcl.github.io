Vue-cli项目结构详解

发表于 2017-05-15   |   分类于 技术   |  

vue-cli脚手架工具为我们搭建了开发所需要的环境，为我们节省了很多精力。正所谓磨刀不误砍柴工，在开发项目之前很有必要对脚手架工具进行熟悉，我们先来熟悉项目的结构。

项目结构
package.json详解

package.json文件是项目根目录下的一个文件，定义该项目开发所需要的各种模块以及一些项目配置信息（如项目名称、版本、描述、作者等）。

1、package.json 里的scripts字段，这个字段定义了你可以用npm运行的命令。在开发环境下，在命令行工具中运行npm run dev 就相当于执行 node build/dev-server.js .也就是开启了一个node写的开发行建议服务器。由此可以看出script字段是用来指定npm相关命令的缩写。

2、dependencies字段和devDependencies字段

  • dependencies字段指项目运行时所依赖的模块；
  • devDependencies字段指定了项目开发时所依赖的模块；

在命令行中运行npm install命令，会自动安装dependencies和devDempendencies字段中的模块。

.babelrc文件

Babel解释器的配置文件，存放在根目录下。Babel是一个转码器，项目里需要用它将ES6代码转为ES5代码。如果你想了解更多，可以查看babel的知识。

.editorconfig文件

该文件定义项目的编码规范，编译器的行为会与.editorconfig文件中定义的一致，并且其优先级比编译器自身的设置要高，这在多人合作开发项目时十分有用而且必要。
